# CTF Writeup: Ghost Mantis LCG PRNG

## Challenge Description
**Context:** Ghost Mantis generated session secrets using a custom PRNG. Several tokens and partial timestamps were recovered. Predict the next valid token.
**Flag Format:** `CREST{next_token}`

### Provided Files:
1. `prng.txt`: Contains the pseudo-random number generator algorithm.
2. `timestamp.txt`: Contains a hint about when the tokens were generated (2025-02-15 between 10:30:00 and 10:30:20 UTC).
3. `tokens.txt`: Contains a sequence of 5 generated tokens.

---

## Analysis

### 1. The PRNG Algorithm (`prng.txt`)
```text
state = seed
state = (state * 1103515245 + 12345) mod 2^32
token = (state >> 12) & 0xFFFFF
```
This is a classic **Linear Congruential Generator (LCG)**. 
- The multiplier is `1103515245`
- The increment is `12345`
- The modulus is `2^32` (meaning the state is a 32-bit integer)
- The output token is generated by shifting the state right by `12` bits and masking with `0xFFFFF`, producing a 20-bit token.

### 2. The Tokens (`tokens.txt`)
```text
Generated tokens:
0xc7a1
0xd4d8c
0x14bdf
0x7ed1e
0xac42f
```
We have 5 consecutive 20-bit tokens. 

### 3. The Vulnerability
The LCG state is strictly 32 bits (from `mod 2^32`). A 32-bit space (`4,294,967,296` possible seeds) is small enough that a modern CPU can brute-force the entire state space in a few seconds.

We are given 5 tokens, which represents $5 \times 20 = 100$ bits of information about the state. Since the state itself is only 32 bits, the sequence of 5 tokens is more than enough to uniquely identify the original 32-bit seed. We don't even need the exact timestamp to filter the results, although it provides a good starting point or sanity check.

---

## The Exploit (Brute-Force)

We can write a fast C program to iterate over all possible 32-bit seeds, run the LCG forward for 5 steps, and check if the generated tokens match the provided sequence. Once a match is found, we run the LCG one more time to predict the 6th token.

```c
#include <stdio.h>
#include <stdint.h>
#include <stdlib.h>

#define MULT 1103515245
#define INC 12345

// Advance the LCG state
uint32_t next_state(uint32_t state) {
    return (uint32_t)((state * MULT + INC) & 0xFFFFFFFF);
}

// Extract the 20-bit token from the state
uint32_t get_token(uint32_t state) {
    return (state >> 12) & 0xFFFFF;
}

int main() {
    uint32_t tokens[] = {0xc7a1, 0xd4d8c, 0x14bdf, 0x7ed1e, 0xac42f};
    
    printf("Searching 32-bit space...\n");
    // Iterate over all possible 32-bit seeds
    for (uint64_t seed = 0; seed < 0x100000000ULL; seed++) {
        uint32_t s = (uint32_t)seed;
        int ok = 1;
        
        for (int i = 0; i < 5; i++) {
            s = next_state(s);
            if (get_token(s) != tokens[i]) {
                ok = 0;
                break;
            }
        }
        
        // If all 5 tokens match, we found the right seed
        if (ok) {
            printf("Found seed: %u\n", (uint32_t)seed);
            
            // Generate the next state and corresponding token
            s = next_state(s);
            printf("Next token: %x\n", get_token(s));
            return 0;
        }
    }
    printf("Not found.\n");
    return 0;
}
```

### Execution
Compiling and running the program with optimizations (`-Ofast`):
```bash
gcc solve.c -Ofast -o solve
./solve
```

**Output:**
```
Searching 32-bit space...
Found seed: 1739538000
Next token: a67fe
```

The script successfully checked the 32-bit space and found the initial seed `1739538000` (which is a UNIX timestamp corresponding to roughly `2025-02-14 13:00 UTC` - close to, but slightly earlier than the hint, possibly suggesting the server's time was misconfigured or using a different timezone). 

The 6th token derived from the state is `a67fe`.

## Flag
**CREST{0xa67fe}**
